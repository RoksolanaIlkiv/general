Parent component with Default change detection strategy

This data will display in parent component only: {{ data() }}
Parent data: {{ signalParentData() }}

Explanation

Whenever change detection is triggered, we'll see a notification explaining where the changes are happening.

With OnPush change detection method, the component will only be checked for changes(In our case) if input reference changes or changes happening in component itself.
With Default change detection method, the component will be checked if any changes happened in parent component as well.
  1. If you click 'Update only parent data', changes are made in the parent component only, but also triggers detection in a second child component.
  2. 'Update parent data' will change parent data 1 time. It will trigger changes in first child only ones, but each time for second child.
  3. 'Change parent data' will change data each time. It will trigger changes in both components.
  4. When 'Update child data' clicked in first child it will trigger changes for parent and second child.
  5. When 'Update child data' clicked in second child it will trigger changes for parent, but not for first child.
