Explanation

Hierarchical dependency injection enables you to share dependencies between different parts of the application only when and if you need to. When it checks for available suppliers, it moves from the child up. So, if the child component has no providers, it will use it from the closest parental and only than from the root.

This behavior can be adjusted using resolution modifiers.
  1. For RootService we create one instance of the service(singleton) and use it. So it will be same in any part of our application.
  2. For ParentService, the variable was written at the parent component level. So will also be available to all children.
  3. But for ChildService it provided in parent component and child component. Since the component will use the closest available value, we will see different results for the child and parent components.
